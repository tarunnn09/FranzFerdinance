# Franz Ferdinand — Artist Portfolio Website

A multi-page artist portfolio website I built during my **1st year of college (Semester 2)**. This was honestly one of the more challenging and fun projects I worked on — designing something that actually *looks* like a real artist's website rather than a generic template took a lot of iteration.

🌐 **Live Site:** [https://tarunnn09.github.io/FranzFerdinance/](https://tarunnn09.github.io/FranzFerdinance/)



## What This Project Is About

Franz Ferdinand is a fictional experimental artist based in the Netherlands, known for creating kaleidoscopic and perception-bending visual art. I designed and built the entire website for this character — from the landing page to the gallery and shop — as a way to practice building real-world, production-style websites.

The goal was simple: make something that looks and feels like a website a professional artist would actually use.



## Pages

- **Home** — Hero section with featured artwork and a strong visual introduction
- **About** — Artist bio, exhibitions, publications, and collections
- **Gallery** — A dedicated page showcasing the artwork
- **Shop** — A page for browsing and purchasing artwork
- **Contact** — Get in touch section with social links



## What I Focused On (Things Worth Looking At)

If you're a recruiter or someone reviewing the project, here's what I'd point you toward:

- **Multi-page structure** — Not just a single scrolling page. Separate HTML files for gallery, shop, and main page with consistent navigation across all of them.
- **Responsive Design** — The layout adapts across screen sizes. I paid attention to how things look on mobile vs desktop since most visitors aren't on a desktop.
- **Clean, semantic HTML** — Proper use of sections, nav, headings, and structure rather than throwing everything into divs.
- **CSS layout techniques** — Used Flexbox and Grid for layout. The gallery and shop sections in particular involved some thoughtful layout work.
- **Visual design decisions** — Chose fonts, spacing, and color scheme deliberately to match the "experimental artist" aesthetic. Design isn't an afterthought here.
- **Image optimization** — Images are served in `.webp` format for faster load times, which shows awareness of web performance basics.
- **Hover effects & transitions** — Small details like smooth transitions on buttons and links. These add up to a much more polished feel.
- **Sticky / functional navigation** — Navigation works correctly across all pages and links to the right sections.
- **GitHub Pages deployment** — The site is live and publicly accessible, not just sitting on my machine.



## Built With

- HTML5
- CSS3
- Vanilla JavaScript
- Hosted on GitHub Pages

No frameworks, no libraries — everything from scratch. I wanted to make sure I actually understood what was happening before reaching for shortcuts.



## What I Learned

This project taught me a lot more than just writing HTML and CSS. Getting the layout to behave consistently across different screen sizes was frustrating at first but clicked once I got more comfortable with Flexbox. I also learned how to properly structure a multi-page site and keep the navigation synced without repeating myself too much.

Deploying to GitHub Pages was also new for me — setting up the repo, pushing changes, and watching them go live was genuinely satisfying.
